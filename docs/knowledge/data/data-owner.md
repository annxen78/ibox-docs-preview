# Владелец данных 
Владелец данных — это сотрудник, который отвечает за конкретную информацию в системе.
<img src="/img/knowledge/data/owner.png" alt="Владелец данных" width="500" />

*Например,* **владелец данных**
- *В карточке клиента* — это менеджер, который с ним работает
- *В прайс-листе* — тот, кто может менять цены
- *В товаре* — сотрудник, который отвечает за описание, остатки и фото

Обычно владелец может редактировать данные, но точные права зависят от настроек доступа.

## Зачем это нужно
Если в ролях указано ограничение *только свои*, сотрудник сможет:
- Видеть только свои записи
- Редактировать и удалять только то, что ему принадлежит

<img src="/img/knowledge/data/data-role.png" alt="Настройки роли" width="400" />

Это помогает защитить данные и разделить ответственность между сотрудниками.
